How to use it

  • Just say show my kanban / open my todo — the AI calls open_component to render it.
  • Say what data do I have — the AI uses data_collections to list every collection; open the dashboard for an overview.
  • Want a tool that doesn't exist? Say make me an X — the AI reads the authoring guide, writes it, saves it to your local registry, and reuses it forever.
  • One component can bind to different data: open my work collection with kanban.
  • Let the AI operate for you: mark "research the spec" done — any open widget auto-refreshes.

Where the data lives

All in one SQLite file on your machine (open-mcp-app.db), with a full append-only change history. Delete that file to reset everything.

Mental model

Components are just the "skin" (reusable views); the data is the substance (collections). You and the AI edit the same data — close the chat and it's still there.
